# Insight 계층 가이드

## 개요

Insight는 Deep Work Quality Gate 시스템의 3번째이자 마지막 계층이다.

| 계층 | 마커 | 동작 | 용도 |
|------|------|------|------|
| Required | ✅ | 실패 시 차단 | 필수 검증 (테스트, 타입체크) |
| Advisory | ⚠️ | 실패 시 경고 | 설계 품질 (SOLID 리뷰) |
| **Insight** | **ℹ️** | **결과 기록만** | **코드 메트릭, 복잡도, 의존성** |

Insight 게이트는 **절대** 워크플로우를 차단하지 않는다. 모든 결과는 정보 제공 목적으로만 기록된다.

## 내장 분석 4종

`/deep-insight` 명령어는 4종류의 분석을 자동으로 수행한다.

### A. 파일 메트릭

각 파일의 코드 줄 수, 함수 수, export 수를 측정한다.

**활용**: 전체 코드 규모 파악, 파일 간 크기 불균형 발견

### B. 복잡도 지표

- **대형 파일** (300줄 초과): 분리 대상 후보
- **장함수** (50줄 초과): 리팩토링 대상 후보
- **깊은 중첩** (4단계 초과): 가독성 저하 지점

**활용**: SRP 위반 가능성이 높은 코드 식별. Advisory 계층의 SOLID 리뷰와 연계하면 더 정확한 설계 판단 가능.

### C. 의존성 분석

- **순환 참조**: 모듈 간 양방향 의존은 변경 파급 범위를 넓힘
- **Hub 파일**: 5곳 이상에서 참조하는 파일은 변경 시 영향 범위가 큼
- **Import 깊이**: 의존 체인이 깊을수록 디버깅이 어려움

**활용**: 아키텍처 결합도 파악, 리팩토링 우선순위 결정

### D. 변경 요약 (워크플로우 전용)

`file-changes.log`(PostToolUse 훅 자동 기록) 또는 `git diff --stat` 기반으로:

- 파일별 수정 횟수
- 추가/삭제 줄 수
- 신규 생성 vs 수정 비율

**활용**: 구현 과정에서 어떤 파일이 가장 많이 변경되었는지 파악

## 커스텀 Insight 게이트 정의

`plan.md`의 Quality Gates 테이블에 ℹ️ 마커로 추가한다:

```markdown
## Quality Gates
| Gate | 명령어 | 필수 | 임계값 |
|------|--------|------|--------|
| Type Check | `npx tsc --noEmit` | ✅ | — |
| SOLID Review | [auto] | ⚠️ | — |
| Complexity | `npx complexity-report src/` | ℹ️ | — |
| Bundle Size | `du -sh dist/` | ℹ️ | — |
```

- ℹ️ 게이트의 임계값은 참고용 — 초과해도 차단하지 않음
- 명령어 실행 실패도 워크플로우에 영향 없음 (SKIP으로 기록)

## 분석 한계

- **언어별 정확도**: grep 기반 휴리스틱이므로 언어에 따라 함수 수 등이 부정확할 수 있음
- **순환 참조 탐지**: 내부 모듈만 대상. 외부 패키지를 통한 간접 순환은 감지하지 않음
- **파일 수 제한**: 50개 초과 시 상위 50개만 분석
- **정밀도**: 정보성 도구이므로 정밀한 정적 분석 도구(ESLint, SonarQube 등)를 대체하지 않음
